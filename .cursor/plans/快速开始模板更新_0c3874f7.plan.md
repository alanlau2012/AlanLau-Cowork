# 快速开始模板更新计划

## 变更内容

将当前 3 个通用模板（文件夹整理、数据分析、批量文件处理）替换为原型中的 6 个业务场景模板：

| 模板 | 领域 | 描述 |

|------|------|------|

| 跨厂商配置翻译 | NIS | 旧网/异构网配置导出翻译 |

| LLD实时校准 | NIS | 详细设计与现网数据对比 |

| 验收件自动纠错 | SEC | PAC/FAC截图自动审核 |

| 故障证据链梳理 | AMS | 多源日志整合生成时间轴 |

| SOC规范应答 | 交付 | 技术规范书逐条应答 |

| 分包商影子审计 | SEC | 分包商操作日志扫描 |

## 修改文件

### 1. [renderer/index.html](renderer/index.html)（第 66-83 行）

更新模板卡片 HTML：

- 从 3 个卡片扩展为 6 个（2行3列布局）
- 每个卡片添加领域标签 `<span class="domain-tag nis">NIS</span>`
- 更新 `data-template` 属性为新的模板 ID

### 2. [renderer/style.css](renderer/style.css)（第 1694-1742 行附近）

添加领域标签样式：

```css
/* 领域标签颜色变量 */
--tag-nis: #3b82f6;      /* 蓝色 - NIS */
--tag-nis-bg: #eff6ff;
--tag-sec: #10b981;      /* 绿色 - SEC */
--tag-sec-bg: #ecfdf5;
--tag-ams: #f59e0b;      /* 橙色 - AMS */
--tag-ams-bg: #fffbeb;
--tag-delivery: #8b5cf6; /* 紫色 - 交付 */
--tag-delivery-bg: #f5f3ff;

/* 领域标签样式 */
.domain-tag { ... }
.domain-tag.nis { ... }
```

调整卡片样式：

- `.template-card` 改为 `position: relative; text-align: left;`
- `.templates-grid` 改为固定 `grid-template-columns: repeat(3, 1fr);`

### 3. [renderer/uiHelpers.js](renderer/uiHelpers.js)（第 281-289 行）

更新 `getTemplateContent()` 函数，替换模板 ID 和提示词内容：

```javascript
const templates = {
  'config-translate': `请帮我进行跨厂商配置翻译...`,
  'lld-calibration': `请帮我进行LLD与现网数据校准...`,
  'pac-review': `请帮我审核验收交付件...`,
  'fault-evidence': `请帮我梳理故障证据链...`,
  'soc-response': `请帮我应答技术规范书要求...`,
  'subcon-audit': `请帮我审计分包商作业质量...`
};
```

## 验证步骤

1. 启动应用 `npm start`（两个终端）
2. 确认首页显示 6 个模板卡片（2行3列）
3. 确认每个卡片右上角有正确的领域标签颜色
4. 点击每个模板，验证提示词正确填充到输入框
